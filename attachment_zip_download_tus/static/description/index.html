1.Configuration

To use this feature in any module you first need to configure it in server action. Follow steps to easily configure it.
First enable developer mode from the general setting.
Go to setting --> Technical --> Action --> server action --> search for default action "zip file" created by us
Fill in the blanks with the information provided below.

Action name - Name of the action
Model - change model name according to your need
Type - Always select Zip File here

After filling details click on “Create Contextual action” to activate it.

2.See Action

Open the module to view the zip file action, for example we have created it for Sale order.
To view it go to Sales module --> order --> sale order --> select orders --> click on action
Server Actions have been put here.

3.Attach file

Whatever files you include in the attachment, they will all be zipped when the zip is generated.
This image shows an attachment icon and an "Attach file" button. Please attach your file here.

4.Create Zip

To create a zip, first select the records from which you want to create the zip. As previously said, zipped files
will be created from the files that are available in the attachment, so make sure you have attached the files in
the attachment. For example, if you selected ten entries and any or all of them have an attachment, the warning will not occur. But,
if all ten attachments are empty, an warning will be displayed.

1.Can we zip files such as invoices, bills, credit notes, quotations, sales orders, and so on?

Ans. No, but if you want to do then attach those files in attachment.

2.Do I need to create Server action for all models?

Ans. Yes. You must first create a Server Action for the model you want to enable zipping.ex. Sale order, invoice, contact etc.

3.How to create server action?

Ans. First activate developer mode, then go to Setting→Technical→Action→Server action→Create
